# codexU v1.1.3

本版本修正 Codex 官方用量口径，并完成 OpenClaw 任务日期、摘要、进度和活跃度排序修复。

## Codex 用量修正

- “最近一天 / 近 7 天 / 累计”和用量趋势直接使用 Codex App `account/usage/read` 的官方服务端日桶，与 ChatGPT 个人资料的 token activity 使用同一来源。
- 本机 `token_count` 明细保留为“本机原始上下文（非官方用量）”，只用于缓存/输入/输出拆分、项目归因和 API 等效价值。
- `--dump-json` 输出新增官方趋势及每日桶，方便逐日核验数据来源。
- 空白热力图日期会按数据源显示“无官方记录”或“无本机记录”，不再混淆口径。

本机验证样本中，2026-07-16 官方值为 `457,746,130`，本机原始上下文为 `3,003,313,840`，其中缓存输入为 `2,836,447,744`。两者是不同指标，官方没有公开换算公式，因此本版本不做推测性折算。

## OpenClaw 任务修正

- 支持 `yyyy-MM-dd HH:mm` 创建时间，不再因解析失败回退到截止日期。
- 详情页分别标注创建时间和截止时间；只有日期的截止值不再显示虚假的 `00:00`。
- 排序只使用可信的最近活动或创建时间，未来截止日期不会把长期不活跃任务顶到前面。
- 任务摘要会整理负责人、行动、KPI、风险、备选和补充信息，并清理重复前缀。
- 详情页新增完成进度条：优先使用明确进度字段，完成状态显示 100%，未知进度显示 `--`，不估算不存在的数据。
- 保持任务卡点击打开详情、Codex 线程深链和其他 Agent 行为不变。
